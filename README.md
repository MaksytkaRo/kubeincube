# kubeincube (will be changed)
Agentic AI written in Go, for infrastructure purposes

# Quis custodiet ipsos custodes?
The same is appliable to controlling applications. Many of companies need better tools to manage internal communication of Developers, Operators and ~~machines~~ other bees in the hive.
This project targets on building PaaS by LLM and idea of chat-bot, delighted by third-order cybernetics.

# Our capabilities
To make a program, which will be possible to interact with known chats, receive messages from there, analyze and push to LLM, in general. If there is access to host's, it running, tools, then it will be able
to execute some programs, and our goal is to understand, how to reach this effectiveness it can give.

# Warning
This project want to be widely used in corporative scenarios, so not only ad-hoc for developer.
